# 對話紀錄 2026-04-08

## 主題

AI Memory Vault v3 — Windows Scheduler 重構 + vault-menu.ps1 排程管理 UI

## 關鍵決策

1. **不使用 APScheduler 守護模式**：改為純 Windows Task Scheduler，`vault-scheduler.exe` 預設 `--once` 模式
2. **管理員提權策略**：不在 installer 強制 UAC，改為選單進入 2/3 選項時偵測並提示用戶重開
3. **顯示完成結果**：`--once` 模式執行完後印出 ✅/❌ 結果清單，等 Enter 再關閉（代替靜默執行）

## 問題與解法

| 問題 | 解法 |
|------|------|
| 選項 3 進去直接啟動守護模式 | 重寫子選單為完整管理 UI（6 選項） |
| `Register-ScheduledTask` 需要管理員 | `$isAdmin` 偵測 + `Start-Process -Verb RunAs` 提權重啟 |
| torch DLL 在 EXE 啟動崩潰 | auto_scheduler/scheduler/vault 三個檔案的 top-level import 改為 lazy |
