# 💬 AI 對話紀錄 — 2026-04-04 Phase 12

## 對話主題

多組織架構修正 + 環境設定工具建置

## 關鍵決策

1. **`organization: str` → `organizations: list`**
   - 原因：一個人可同時屬於多個組織，單一字串是錯誤的語意設計
   - 影響：config.py / server.py / main.py 三處同步修改
   - 向後相容：ConfigManager.load() 加 migration shim

2. **組織管理 UI 設計迭代**
   - 第一版（toggle 切換式）→ 被否決：已設定的就是你的，不需要再「選擇加入」
   - 最終版（新增/移除分離）→ 新增從 Vault 掃描 + 輸入；移除可選保留目錄

3. **環境設定工具架構**
   - `環境設定.bat` → `setup-menu.ps1` → `main.py --setup-section`
   - 首次無 config.json → 自動完整引導
   - 後續 → 區段選單（vault / user / org / llm）
   - `--reconfigure` = 依序 1→4 引導，不重建 DB

4. **UX 統一原則**
   - 所有 input() 提示：`xxx（Enter 保持不變）[目前值]:`
   - 每個區段畫面：顯示目前值 → 提問 → 儲存

## 遇到的問題與解法

| 問題 | 根因 | 解法 |
|------|------|------|
| setup-menu.ps1 語法爆炸 | PS5.1 預設 CP950 讀 UTF-8 中文 | 加 UTF-8 BOM |
| 環境設定.bat 中文亂碼 | cmd 預設 CP950 | 加 `chcp 65001` |
| _run_setup_wizard 使用已移除欄位 | 舊 `organization: str` | 改用 `organizations: list` |

## 修改的檔案

- `config.py` — UserConfig 欄位 + load() migration shim
- `mcp_app/server.py` — _provision_org_skeleton() 迭代 list
- `main.py` — 大幅重構：新增 --setup-section / --reconfigure / 4 個區段函式 / _run_full_reconfigure / org wizard 重寫
- `setup-menu.ps1` — 新建，PowerShell 互動選單
- `環境設定.bat` — 新建，雙擊啟動入口
