# 🤖 AI 對話紀錄 — 2026-04-04 Phase 13

## 對話主題

Vault 結構重構：`todos.md` + `handoff.md` 合併為 per-project `status.md`

## 問題背景

使用者觀察到 `_config/` 下的 `todos.md` 與 `handoff.md` 內容高度重疊，且隨著專案增加，全域單一檔案無法擴展。

核心問題：
- `todos.md` 的「完成/進行中/待處理」與 `handoff.md` 的「完成項目/待續事項」完全重複
- 同時做多個專案時，全域檔案變成多專案混雜的大雜燴
- 各專案的 VS Code workspace 無法只讀取自己的狀態

## 關鍵決策

| 決策 | 原因 |
|------|------|
| 廢棄全域 `todos.md` | 是 `handoff.md` 的子集，無獨立存在價值 |
| 廢棄詳細版 `handoff.md` | 多專案時內容衝突，難以維護 |
| 引入 per-project `status.md` | 待辦 + 工作脈絡 + 決策三合一，每專案獨立 |
| `_config/handoff.md` 降格為索引 | 只記「上次動了哪些專案 + 連結到 status.md」 |

## 資料流設計

```
收工寫入：各專案 status.md → _config/handoff.md 索引 → personal/reviews/daily/
開工讀取：_config/handoff.md → 活躍專案各自的 status.md
```

## 修改檔案

### 程式碼
- `config.py` — `VaultPaths.proj_status = "status.md"` + `project_status_file()` helper + `get_initial_files()` 移除 `todos.md`
- `services/setup.py` — `_gen_nav()` 更新、`_gen_handoff()` 改為輕量索引、移除 `_gen_todos()`
- `mcp_app/server.py` — instructions label 調整

### Vault 資料
- 新建 `workspaces/LIFEOFDEVELOPMENT/projects/ai-memory-vault/status.md`（現有資料遷入）
- 改寫 `_config/handoff.md`（輕量索引格式）
- 改寫 `_config/nav.md`（加 status.md 描述）
- 改寫 `_config/end-of-day-checklist.md`（收工步驟 4+5 更新）
- 刪除 `_config/todos.md`
