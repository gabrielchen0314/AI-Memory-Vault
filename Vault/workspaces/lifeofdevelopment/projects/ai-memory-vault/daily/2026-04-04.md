## 遇到的問題

1. `cleanup="incremental"` vs `"full"` 語意差異：
   - `incremental`：只處理本次傳入的文件（不刪不在清單的舊向量）
   - `full`：以本次掃描為完整快照，刪除不在清單的所有向量
   - **結論**：全量 sync 時必須用 `full`，單檔 sync_single 繼續用 `incremental` 是正確的

2. Hybrid BM25+Vector EnsembleRetriever RRF 排名問題：
   - DB corpus 縮小後，無語意相似度的唯一 marker token 被 Vector 排低
   - BM25 rank=1，Vector rank=N/A → RRF 組合後排到第 6（top_k=5 截斷）
   - 解法：精確 token 查詢使用 `iMode="keyword"`
