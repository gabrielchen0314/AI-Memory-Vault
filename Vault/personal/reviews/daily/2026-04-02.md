## 遇到的問題

- PowerShell 5.1 繁中 Windows 讀取 UTF-8 無 BOM 檔案時，預設用 CP950（ANSI），導致中文字元被誤判為語法錯誤
  - **解法**：用 `New-Object System.Text.UTF8Encoding $true` 寫出帶 BOM 的 UTF-8 檔案
- `replace_string_in_file` 工具遇到舊內容仍存在尾部，導致 PS1 同時含新舊兩段程式碼
  - **解法**：用 `$content.IndexOf()` 找切割點後截斷

## 筆記 / 學到的事

- **工作回顧排程的限制**：`daily-review` 排程只建立空白模板，不會自動記錄 VS Code 對話內容
  - 若要自動記錄對話，需等 Phase 2 `ChatService` 完成後，在每次對話結束時呼叫 `log_conversation()`
- **PowerShell 5.1 BOM 規則**：
  - UTF-8 無 BOM → Windows 預設 ANSI 讀取 → 中文字元破損
  - UTF-8 BOM (EF BB BF) → PowerShell 正確識別 UTF-8
  - 解法：`[System.IO.File]::WriteAllText(path, content, New-Object System.Text.UTF8Encoding $true)`
